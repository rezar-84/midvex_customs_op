1

Setup & Security Configuration

Before general users can access the application, an administrator must assign appropriate access roles to each user. The module defines four linear user groups:

Group / Role Inherits Rights From Permissions & Capabilities
Customs User — Can create and edit Customs Files, add product lines, upload compliance document attachments, and log chatter messages.
Customs Document Approver Customs User Can mark documents as under review, approve, reject, or request correction with mandatory notes.
Customs Manager Customs Document Approver Can override shipment readiness blockages (with audit log), close/cancel operations, and view reports.
Customs Administrator Customs Manager Full access to master configuration (workflow stages, document types, and security rules).

Role Assignment Steps

Navigate to Settings > Users & Companies > Users. Select the user, locate the Customs Operations category in the access rights tab, choose their group, save, and refresh the browser.

2

Master Data Configuration

1. Workflow Stages

Customs Files advance through stages representing shipment progress. Administrators can configure stages globally or per company:

  • Stage Name: The label displayed in the Kanban and form views (supports multi-language translations).
  • Sequence: Integer representing order (smaller sequences display first).
  • Folded in Kanban: Check this to collapse the Kanban column by default.
  • Closed/Cancelled Stage: Defines terminal stages for closed shipments.
  • Company: Leave blank for global stages, or restrict to a specific company.

2. Document Types

Document types define compliance requirements (e.g., Certificate of Analysis, Health Certificate):

  • Code: Unique identifier (e.g., COA, HC, COO).
  • Turkish Name: Local translation label.
  • Default Responsible Party: Usually Supplier, Broker, or Internal.
  • Default Requirement Level: Mandatory (blocks shipment if missing) or Optional.
  • Original Normally Required: Check if physical copy is legally required.
3

Creating & Managing Customs Files

  • 1

    Initialize the Customs File

    Navigate to Customs Operations > Operations > All Customs Files and click New. Fill in the priority and responsible employee. Saving generates a unique reference ID following the sequence CUS/YYYY/NNNN.

  • 2

    Link Parties and Orders

    In the Overview tab, select suppliers, broker, forwarder, and carrier. Select related Purchase Orders and Incoming Shipments. The supplier selection will validate and restrict options on save to ensure data consistency.

  • 3

    Record Products and Batches

    Under the Product Lines tab, click Add a line to add batch-level details including quantity, net weight, gross weight, production date, and expiry date. Weight validation prevents net weight from exceeding gross weight.

Product Line Lock Rule

To preserve verified data integrity, product lines become completely read-only and locked once the Customs File progresses to the "Document Review" stage or beyond.

4

Compliance Document Tracking

  • 1

    Define Document Requirements

    Under the Required Documents tab (or via the header Smart Button), define the list of required compliance records. Setting Blocking Requirement to True makes this document critical for shipment readiness.

  • 2

    Upload Attachments

    Click a line to open its form and upload the copy using standard attachments. The system automatically manages and increments the Version Number each time a new attachment is added.

  • 3

    Verification & Approval

    Document Approvers inspect the attachment and update the status. If approved, the document is complete. If rejected, a rejection reason must be recorded, which triggers corrective alerts in the chatter.

5

Shipment, Customs Entry & Handover

Logistics & Transport

Record essential shipping information under the Shipment Details tab:

  • Departure/Destination countries
  • Planned & Actual departure dates
  • Container and Booking numbers
  • Transport documents (Bill of Lading / Air Waybill)

Customs & Laboratory

Record compliance and clearance results under the Customs & Laboratory tab:

  • Customs declaration number and filing date
  • Inspection/Laboratory requirements and results
  • Customs release date and warehouse delivery confirmation
6

Closing the Customs File

When the shipment is safely received at the warehouse, the Customs Manager moves the file's stage to Closed.

Closing Validation Rules

If mandatory document requirements remain incomplete, or if critical activities are left open, Odoo blocks the stage change with a validation error. If customs reference details or actual warehouse receipt dates are missing, the system posts a warning in the chatter but allows closing to preserve operational flexibility.
