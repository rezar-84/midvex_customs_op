# Acceptance Criteria and Review Checklist

## 1. Installation & Upgrade
- Installs on a clean Odoo 19 database without traceback errors.
- Upgrades cleanly on an existing database containing previous standalone Customs Operations data without breaking existing records.
- All new database constraints compile and load successfully.

---

## 2. Purchase Order Integration
- **Manual Trigger**: Clicking **Create Import Operation** on a PO:
  - If no operation exists: Creates one Customs File, syncs all headers and product lines, and opens the form view.
  - If an operation already exists: Opens the existing Customs File (no duplicate is created).
- **Auto-Creation**: Confirming a PO auto-creates a Customs File only if:
  - `customs_required` is True, OR
  - `is_import_purchase` is True (vendor country is different from company country), OR
  - Any PO line product/category has compliance flags set.
- **Auto-Creation Exclusion**: Confirming a normal domestic PO (vendor country same as company country) with no flags does NOT create a Customs File.
- **PO Header Sync**: Mapped fields (Vendor, Company, Currency, Incoterm, Expected Date, Total Amount) are correctly synced.
- **PO Line Sync**:
  - `customs.operation.line` is created for each product line, correctly linking to `purchase.order.line`.
  - Section and note lines are ignored.
  - Clicking **Sync Purchase Lines** refreshes lines, updates quantities, and appends new lines without losing manually reviewed line-level customs data.

---

## 3. Stock Receipt (Inventory) Integration
- All stock receipts (incoming pickings) generated by standard PO confirmation are automatically linked to the Customs File's `picking_ids`.
- Validating an incoming picking shows a warning message or logs an alert in the picking chatter if the Customs File is not yet cleared (stages before Customs Cleared).
- If strict mode `customs_block_receipt_before_clearance` is set to `True`, the receipt should block validation, whereas if `False` (default), validation should succeed with a warning.
- Linked pickings are visible via smart buttons on both the picking form and the Customs File form.

---

## 4. Vendor Bills & Costs Integration
- Vendor bills created from the linked Purchase Order are automatically associated with the Customs File.
- Users can manually associate external expense bills (freight, duty, broker fees) to the Customs Operation.
- Smart buttons display correct counts and link back and forth between bills and operations.

---

## 5. Product Customs Profile Defaulting
- Extending product templates allows setting hs_code, country_of_origin_id, and requirement flags.
- Adding a product to a PO line and syncing it to a Customs File automatically triggers document requirements (such as Health Certificate, COA, or Import Permit) based on these defaults.
- Prevents duplicate document requirements if the type has already been generated.

---

## 6. Security & Multi-Company Constraints
- **Company Consistency**: Enforce database constraints or validations preventing linking a PO, picking, vendor bill, or partner belonging to a different company than the Customs Operation.
- **ORM Access**: Users without Purchase or Accounting privileges can use Customs Files but cannot bypass Odoo security to view financial or PO records via smart buttons.
- Standalone creation by normal users still works within their permitted company bounds.

---

## 7. UI/UX Specifications
- **Purchase Order**:
  - Smart button displays the count of linked Customs Operations.
  - Form shows a clear summary of linked customs state, document completion percentage, readiness, and red warning text for blocking reasons.
- **Customs Operation**:
  - Smart buttons navigate to linked POs, pickings, bills, and sales orders.
- **Stock picking**:
  - Form shows a smart button and a text/badge summary of the customs status.

---

## 8. Automated Tests Coverage
The test suite must verify:
- PO confirmation auto-creation under correct conditions.
- Domestic PO confirmation bypass (no operation created).
- Duplicate prevention (opening existing operation).
- Line item mapping (excluding note/section lines).
- PO line synchronization button behavior.
- Stock picking linkage and validation warning.
- Vendor bills auto-linkage.
- Product template flags triggering compliance document requirements.
- Multi-company access and company consistency validations.

---

## 9. Definition of Done
- All integration hooks, models, and views are implemented.
- Database upgrades work.
- Automated tests pass.
- Turkish translation is updated to support new fields, buttons, and warnings.
- README and guides are updated.
