# Automation and Sync Rules

## 1. Reference Generation
- Format: `CUS/YYYY/NNNN`.
- Trigger: Assigned automatically via sequence on first database commit of the `customs.operation` record.

---

## 2. Purchase Order Auto-Creation Hook

### Trigger Event
- A `purchase.order` transitions state to `purchase` (Order Confirmed).

### Rules for Auto-Creation
Odoo will automatically generate a linked `customs.operation` if **no operation is currently linked to the PO** AND at least one of the following is true:
1. `customs_required` is manually set to `True` on the PO.
2. `is_import_purchase` is set to `True` (automatically set if the vendor's country differs from the company's country).
3. Any product in the PO lines has `customs_required = True` on its product template.
4. Any product in the PO lines belongs to a product category with a customs compliance requirement flag.

### Duplicate Prevention
- The auto-creation logic must search `customs.operation` where `purchase_order_ids` includes the current PO.
- If an operation exists, do not create a duplicate.
- If a user clicks **Create Import Operation** manually on a PO, and an operation already exists, redirect the user's view to the existing record instead of creating a new one.

## 3. Purchase-to-Customs Data Synchronization

### Initial Creation Sync
When creating the Customs Operation from a confirmed PO, copy:
- **Header info**: Vendor (`supplier_ids` M2M field), Company (`company_id`), Currency (`currency_id`), Incoterm (`incoterm_id`), PO Reference (linked to `purchase_order_ids`), expected date, total PO amount (`amount_total`).
- **Product Lines**: Import all PO lines as `customs.operation.line` records.
  - Exclude PO lines of type `note` or `section`.
  - Mapped fields: Product, description, quantity, UoM, net/gross weights, HS/GTİP code, and compliance flags (COA, Health Cert, Import Permit) loaded from product template defaults.

### PO Change Sync ("Sync Purchase Lines" trigger)
If a PO is modified (e.g. quantity changed or line added) prior to the operation moving past the `Waiting for Documents` stage:
- Adding a line creates a corresponding `customs.operation.line` linked to the new `purchase.order.line`.
- Changing quantities updates the operational quantities.
- Under no circumstances should sync overwrite manually reviewed and modified fields on the Customs Operation Lines (such as manually entered batch numbers, expiry dates, or weights) unless the user explicitly triggers an overwrite.

## 4. Stock picking Synchronization

### Automatic Linkage
- When stock pickings are generated by Odoo's standard PO confirmation, hook into the creation or confirmation flow to associate the generated `stock.picking` records to the Customs Operation's `picking_ids`.

### Receipt Status Warning
- **Trigger**: A stock picking linked to a Customs Operation is set to state `done` (Validated).
- **Rule**: If the linked Customs Operation stage has `is_closed = False` AND the stage is prior to `Customs Cleared` (e.g., stage is Draft, Waiting for Docs, Doc Review, Correction, Shipped, Arrived, or Customs Clearance):
  - Raise a soft warning message in the Odoo user interface or post an alert in the picking chatter: *"Goods are being received before Customs Operation is cleared."*
  - Log an activity for the purchasing/logistics team to inspect compliance.
  - Do not hard block stock transaction unless strict mode configuration `customs_block_receipt_before_clearance` is enabled.

---

## 5. Billing / Invoice Synchronization

- **Trigger**: A vendor bill (`account.move` of type `in_invoice`) is created with an origin referencing a Purchase Order linked to a Customs Operation.
- **Sync Rule**: Automatically append this vendor bill to the `invoice_ids` field on the linked `customs.operation`.

---

## 6. Document Requirements Auto-Generation

When a Customs Operation is created from a PO, run the Document Requirement Engine:
1. **Always Create (Standard Imports)**:
   - Commercial Invoice (INV)
   - Packing List (PKL)
   - Certificate of Origin (COO)
   - Bill of Lading / Air Waybill (BL / AWB) depending on transport mode.
2. **Product Profile Conditional Triggers**:
   - Create Certificate of Analysis (COA) if any imported product template has `analysis_required = True`.
   - Create Health Certificate (HC) if any imported product template has `health_certificate_required = True`.
   - Create Import Permit (IP) if any imported product template has `import_permit_required = True`.
3. **Company Defaults**: Check for any company-specific requirement templates and append.
